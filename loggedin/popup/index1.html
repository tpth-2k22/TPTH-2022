Problem Statement 1:

Beat Policing Diary(BPD APP) with integration of wanted criminals data.

Problem

Beat Policing is a significant day to day activity which is the rooted base of Police Department. The reports need to be analysed and linked with criminal activity as a proactive measure to combat any offensive activity in a locality. The conventional process of Beat policing has no mechanism to make a background linkage of trail of activity across police station. needs one database having list of wanted criminals with detail particulars so that if any person is arrested in any Police Station, it can be checked into the system instantly whether that person is wanted in any other case throughout the country.

Outcomes

Admin and Interface

Simple and intuitive Mobile App interface to record day to day activity of Beat Policing , ground reporting with audio visual entry.The application should have a desk top admin dashboard to analyse the Beat policing report , duty integrity , reward and performance analysis of individual to PS level

Data Analysis

The individual beat activity to be analysed as a reward score. The collected beat patrolling data to be automatically analysed with the background integration of the existing crime record and instances for automatic inference. This will help in decision support system of interlinking in crime occurrence.

Major Modules

  • Features

    Login based beat patrolling data storage, recording of beat patrolling activity with geo tagging of audio video data upload facility.

  • Interface

    Easy interface to upload data with minimum UI activity.

  • Data

    Case based direct data submission facility.

  • Activity

    Activity reminder and visual cue of actions to be taken.
